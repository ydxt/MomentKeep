# Supabase 实时同步使用指南

## 📋 概述

本文档说明如何在拾光记 (Moment Keep) 中配置和使用 Supabase 实时同步功能。

---

## 🚀 快速开始

### 步骤 1：创建 Supabase 项目

1. 访问 [Supabase](https://supabase.com) 并注册账号
2. 点击 "New Project" 创建新项目
3. 填写项目信息：
   - **Project name**: moment-keep（或任意名称）
   - **Database Password**: 设置一个强密码
   - **Region**: 选择离你最近的区域
4. 等待项目创建完成（约 2 分钟）

### 步骤 2：获取项目凭据

1. 进入项目 Dashboard
2. 点击左侧菜单 **Settings** → **API**
3. 复制以下信息：
   - **Project URL**: `https://xxxxx.supabase.co`
   - **anon/public key**: `eyJhbGc...`（以 ey 开头的长字符串）

### 步骤 3：执行数据库脚本

1. 在 Supabase Dashboard 点击左侧菜单 **SQL Editor**
2. 点击 **New Query**
3. 复制 `docs/supabase_schema.sql` 文件的全部内容
4. 粘贴到 SQL Editor 中
5. 点击 **Run** 执行脚本
6. 确认所有表创建成功（查看输出信息）

### 步骤 4：启用 Realtime 功能

1. 点击左侧菜单 **Database** → **Replication**
2. 确保以下表已启用 Realtime：
   - ✅ todos
   - ✅ habits
   - ✅ habit_records
   - ✅ journals
   - ✅ categories
   - ✅ pomodoro_records
   - ✅ plans
   - ✅ achievements

### 步骤 5：在应用中配置

1. 打开拾光记应用
2. 进入 **设置** → **其他** → **同步设置**
3. 填写配置：
   - **Supabase URL**: 粘贴步骤 2 中的 Project URL
   - **Anon Key**: 粘贴步骤 2 中的 anon key
   - **启用同步**: 打开开关
   - **启用实时同步**: 打开开关（可选）
4. 点击 **测试连接** 确认配置正确
5. 点击 **保存配置**

## 📱 使用同步功能

### 手动同步

1. 在任意数据页面（待办、习惯、日记等）
2. 下拉刷新或点击同步按钮
3. 等待同步完成

### 自动同步

- **实时同步**：启用后，数据变更会立即同步（WebSocket 推送）
- **定时同步**：即使未启用实时，也会按间隔自动同步
- **启动同步**：应用启动时自动检查并同步
- **后台同步**：应用进入后台时自动同步

### 离线使用

- ✅ 离线时可以正常使用所有功能
- ✅ 离线操作会自动加入队列
- ✅ 网络恢复后自动同步离线操作

## 🔧 配置选项

### 同步设置

| 选项 | 说明 | 推荐值 |
|------|------|--------|
| Supabase URL | 项目 API 地址 | 必填 |
| Anon Key | 项目公开密钥 | 必填 |
| 启用同步 | 开启/关闭同步功能 | 开启 |
| 实时同步 | 使用 WebSocket 实时同步 | 开启 |
| 同步间隔 | 定时同步的时间间隔 | 15 分钟 |
| 冲突策略 | 数据冲突时的解决策略 | 服务器优先 |

### 冲突解决策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 服务器优先 | 以服务器数据为准 | 多设备协同 |
| 本地优先 | 以本地数据为准 | 单设备使用 |
| 最后写入 | 比较更新时间，使用最新的 | 默认推荐 |

## 🔍 故障排除

### 连接测试失败

**问题**：点击"测试连接"后显示失败

**解决方案**：
1. 检查 Supabase URL 是否正确（必须以 `https://` 开头）
2. 检查 Anon Key 是否完整复制
3. 确认网络连接正常
4. 确认 Supabase 项目状态正常（Dashboard 可查看）

### 同步失败

**问题**：同步时显示错误

**解决方案**：
1. 查看错误提示信息
2. 检查是否已登录账号
3. 确认数据库表结构正确
4. 检查行级安全策略是否启用

### 实时同步不工作

**问题**：启用实时同步后数据不更新

**解决方案**：
1. 确认 Replication 中已添加所有表
2. 检查 WebSocket 连接状态（查看控制台日志）
3. 尝试关闭并重新开启实时同步
4. 重启应用

### 数据冲突

**问题**：多设备编辑同一条数据导致冲突

**解决方案**：
- 默认使用"最后写入"策略，自动解决
- 如需手动处理，会弹出冲突解决对话框
- 建议避免多设备同时编辑同一条数据

## 🔒 安全说明

### 数据安全

- ✅ 所有通信使用 HTTPS 加密
- ✅ 行级安全策略确保用户只能访问自己的数据
- ✅ 认证 Token 安全存储在设备中
- ✅ 敏感数据（日记内容）额外加密

### 隐私保护

- Supabase 仅存储你的业务数据
- 不会泄露给第三方
- 可随时删除项目和数据

## 🎯 最佳实践

### 1. 首次使用

1. 先在一台设备上配置并同步成功
2. 确认所有数据同步正常
3. 再在其他设备上配置并登录同一账号

### 2. 多设备协同

1. 所有设备登录同一 Supabase 账号
2. 启用实时同步获得最佳体验
3. 避免同时编辑同一条数据

### 3. 离线场景

1. 离线前确保最后一次同步成功
2. 离线操作会自动保存
3. 网络恢复后自动同步

### 4. 数据备份

1. 定期使用导出功能备份数据
2. Supabase 也提供自动备份功能
3. 重要数据建议双保险
