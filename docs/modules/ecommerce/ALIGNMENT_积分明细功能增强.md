## 2. 项目上下文分析

### 2.1 现有项目结构

**技术栈**：
- Flutter 3.0+
- SQLite 本地数据库（sqflite）
- 状态管理：flutter_bloc + flutter_riverpod
- 图表库：fl_chart ^1.1.1（已在 pubspec.yaml 中）
- 日期处理：intl ^0.20.2

**现有相关文件**：
- `lib/presentation/pages/member_level_page.dart` - 会员等级/积分页面，包含积分明细列表
- `lib/services/database_service.dart` - 数据库服务，包含 `getBillItems()` 方法
- `lib/domain/entities/` - 数据实体定义

### 2.2 现有积分明细实现

当前 `member_level_page.dart` 中的积分明细：
- 简单的列表展示
- 按日期降序排序
- 显示标题、日期、积分变化
- 收入/支出用不同颜色区分

**数据模型**（BillItem）：
```dart
{
  'id': String,

```

## 3. 需求理解确认

### 3.1 功能需求分解

基于原始需求，分解为以下核心功能模块：

#### 模块1：时间筛选功能
- 快速筛选：今天、本周、本月、本年
- 自定义日期范围选择
- 日期范围输入验证

#### 模块2：搜索功能
- 按描述/标题搜索积分记录
- 支持模糊搜索
- 实时搜索或点击搜索

#### 模块3：条件筛选功能
- 收支类型筛选：全部、收入、支出
- 交易类型筛选：习惯打卡、待办完成、日记、消费等
- 组合筛选（时间 + 类型）

#### 模块4：收支统计功能
- 周账单统计
- 月账单统计
- 年账单统计
- 自定义时间段统计
- 统计维度：总收入、总支出、净收入、交易次数

#### 模块5：图表展示功能
- 收支趋势图（折线图）
- 收支占比图（饼图）
- 交易类型分布图（柱状图/饼图）
- 支持切换不同图表类型
