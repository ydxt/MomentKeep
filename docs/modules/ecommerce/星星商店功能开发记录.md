#### 1.1 数据库层增强 (product_database_service.dart)

新增了以下推荐相关方法：

##### a) 猜你喜欢推荐
```dart
Future<List<Map<String, dynamic>>> getRecommendedProducts({
  required String userId,
  int limit = 10,
})
```
**实现逻辑**:
- 基于用户浏览历史，分析用户感兴趣的分类
- 从同类别中推荐高销量、高访问量的商品
- 排除用户已浏览过的商品，提供新鲜感
- 如果没有浏览历史，自动降级为热门商品推荐

##### b) 相似商品推荐
```dart
Future<List<Map<String, dynamic>>> getSimilarProducts({
  required int productId,
  int limit = 6,
})
```
**实现逻辑**:
- 首先基于商品标签进行相似度匹配
- 如果标签不足，使用同分类商品作为备选
- 按销量和创建时间排序，确保推荐质量
- 自动去重，避免重复推荐

##### c) 搭配购推荐
```dart
Future<List<Map<String, dynamic>>> getBundleProducts({
  required int productId,
  int limit = 4,
})
```
**实现逻辑**:
- 分析历史订单数据，找出经常一起购买的商品组合
- 统计关联购买频次，按频率排序推荐
- 如果没有订单数据，降级为同分类热门商品
- 支持实物+虚拟商品的混合搭配

##### d) 热门商品查询

### 2. ✅ 商品问答功能

#### 2.1 数据库设计 (product_database_service.dart)

**新建表结构**:
```sql
CREATE TABLE IF NOT EXISTS product_questions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  product_id INTEGER NOT NULL,           -- 商品ID
  user_id TEXT NOT NULL,                 -- 提问用户ID
  user_name TEXT NOT NULL,               -- 提问用户名
  question TEXT NOT NULL,                -- 问题内容
  answer TEXT,                           -- 回答内容
  answered_by TEXT,                      -- 回答者
  answered_at INTEGER,                   -- 回答时间
  is_answered INTEGER NOT NULL DEFAULT 0, -- 是否已回答
  like_count INTEGER NOT NULL DEFAULT 0,  -- 点赞数
  created_at INTEGER NOT NULL,           -- 创建时间
  updated_at INTEGER NOT NULL,           -- 更新时间
  FOREIGN KEY (product_id) REFERENCES star_products(id) ON DELETE CASCADE
)
```

**索引优化**:
- `idx_product_questions_product_id`: 按商品查询
- `idx_product_questions_created_at`: 按时间排序
- `idx_product_questions_is_answered`: 筛选未回答问题

## 二、技术亮点

### 1. 智能推荐算法
- **个性化推荐**: 基于用户行为数据，提供千人千面的推荐
- **多层降级策略**: 
  - 有浏览历史 → 个性化推荐
  - 无浏览历史 → 热门商品
  - 无订单数据 → 同分类商品
- **性能优化**: 
  - 使用SQL原生查询，避免内存计算
  - 异步加载，不阻塞UI
  - 结果缓存（可通过Riverpod进一步优化）

### 2. 数据库设计
- **外键约束**: 商品删除时级联删除相关问答
- **索引优化**: 针对高频查询字段建立索引
- **版本兼容**: 在_initializeDatabase中创建新表，不影响现有数据

### 3. UI/UX设计
- **响应式布局**: 自适应手机、平板、PC不同屏幕
- **加载状态**: 所有异步操作都有loading提示
- **空状态引导**: 无数据时提供明确的操作指引
- **深色主题**: 完美适配深色模式

### 4. 代码质量
- **错误处理**: 所有数据库操作都有try-catch
- **类型安全**: 严格的空值检查
- **代码复用**: 抽取通用组件和方法
- **注释完善**: 关键逻辑都有中文注释
