# 数据存放策略 - 共识阶段

## 1. 明确的需求描述

### 1.1 核心需求

- **数据存储架构**: 建立服务器（cloud目录）和本地存储的分层架构
- **数据分类**: 明确区分必须放在服务器的数据和适合放在本地的数据
- **存储策略**: 制定详细的数据存储策略，参考线上商店和日记系统
- **同步机制**: 设计本地与服务器之间的数据同步策略
- **安全措施**: 确保数据存储的安全性

### 1.2 验收标准

- **目录结构**: 成功创建cloud目录和本地目录结构
- **数据分类**: 所有数据类型都有明确的存储位置
- **同步策略**: 实现数据在本地和服务器之间的同步
- **安全措施**: 敏感数据得到有效保护
- **系统集成**: 存储策略与现有系统无缝集成

## 2. 技术实现方案

### 2.1 存储架构

- **服务器目录** (`用户设置目录/cloud/`):
  - `database/`: 服务器数据库
  - `files/`: 服务器文件存储
  - `logs/`: 服务器日志
- **本地目录** (`用户设置目录/`):
  - `default/`: 默认数据
  - `[用户ID]/`: 每个用户独立目录
  - `local_database/`: 本地数据库

### 2.2 数据分类策略

#### 2.2.1 必须放在服务器的数据

- **用户相关数据**: 用户ID、用户类型、状态、登录记录、信用积分等
- **电商相关数据**: 商品信息、订单数据、交易记录、优惠券等
- **系统相关数据**: 系统配置、日志数据、统计数据等

#### 2.2.2 应该放在本地的数据

- **个人隐私数据**: 日记内容、习惯打卡、待办事项、个人设置等
- **缓存数据**: 商品缓存、图片缓存、搜索历史等
- **离线数据**: 离线商品、离线订单、离线日记等

### 2.3 数据同步策略

- **实时同步**: 用户认证、订单状态、商品库存等关键数据
- **定期同步**: 个人资料、日记数据、习惯记录等非关键数据
- **手动同步**: 大文件、批量数据、备份数据等

### 2.4 数据安全策略

- **服务器数据安全**: 数据加密、访问控制、备份策略、日志审计
- **本地数据安全**: 本地加密、访问保护、数据清理、隐私设置

### 2.5 技术约束

- **无实际服务器**: 使用本地cloud目录模拟服务器
- **存储限制**: 考虑设备存储容量限制
- **离线使用**: 确保在无网络情况下的正常使用

### 2.6 集成方案

- **修改数据库配置**: 将用户和电商相关数据库指向cloud目录
- **更新存储服务**: 修改StorageService以支持服务器文件存储
- **实现数据同步**: 开发数据同步模块
- **添加安全措施**: 实现数据加密和访问控制

## 3. 任务边界限制

### 3.1 范围限制

- **仅制定存储策略**: 不包括具体的代码实现
- **模拟服务器**: 使用本地cloud目录模拟服务器，不涉及实际网络传输
- **现有系统集成**: 确保与现有Flutter项目架构兼容

### 3.2 验收标准

- **文档完整性**: 所有存储策略文档完整且详细
- **技术可行性**: 存储策略在技术上可行
- **业务适配性**: 存储策略满足电商和个人管理系统的需求
- **安全合规性**: 存储策略符合数据安全要求

## 4. 关键假设确认

1. **服务器模拟**: 使用本地cloud目录模拟服务器是可行的
2. **数据同步**: 可以通过本地文件操作实现数据同步
3. **存储容量**: 设备存储容量足够满足需求
4. **安全措施**: 本地加密和访问控制可以有效保护数据
5. **系统集成**: 存储策略可以与现有Flutter项目无缝集成
