# 数据存放策略

## 1. 存储目录结构

### 1.1 服务器目录（cloud目录）

```
C:\Users\Administrator\Documents\MomentKeep\cloud/
├── database/           # 服务器数据库
│   ├── moment_keep_users.db   # 用户主数据库
│   └── moment_keep_ecommerce.db  # 电商数据库
├── files/              # 服务器文件存储
│   ├── products/       # 商品图片/视频
│   ├── users/          # 用户头像/认证文件
│   └── system/         # 系统文件
└── logs/               # 服务器日志
```

### 1.2 本地目录

```
C:\Users\Administrator\Documents\MomentKeep/
├── default/            # 默认数据
│   ├── database/       # 默认数据库
│   └── store/          # 商店默认数据
├── [用户ID]/           # 每个用户独立目录
│   ├── images/         # 个人图片
│   ├── audio/          # 个人音频
│   ├── video/          # 个人视频
│   └── files/           # 其他个人文件
├── local_database/     # 本地数据库
│   ├── moment_keep.db  # 本地应用数据库
│   └── cache.db         # 缓存数据库
└── cloud/              # 模拟服务器目录
```

## 2. 数据存放策略

### 2.1 必须放在服务器的数据

#### 2.1.1 用户相关数据

- **用户主表数据**：用户ID、用户类型、状态、登录记录等
- **买家扩展数据**：昵称、头像、联系方式、会员等级、积分等
- **商家扩展数据**：商家名称、营业执照、联系人信息、店铺信息等
- **管理员扩展数据**：管理员信息、权限等
- **信用积分数据**：买家信用积分、卖家信用积分
- **认证文件**：商家营业执照、身份证等认证材料

#### 2.1.2 电商相关数据

- **商品数据**：商品信息、SKU、规格、库存等
- **订单数据**：订单信息、订单商品、订单状态等
- **交易数据**：支付记录、退款记录、物流信息等
- **优惠券数据**：优惠券信息、用户优惠券等
- **红包数据**：红包活动、领取记录等
- **购物卡数据**：购物卡信息、交易记录等
- **物流数据**：物流公司、物流跟踪等

#### 2.1.3 系统相关数据

- **系统配置**：全局设置、版本信息等
- **日志数据**：操作日志、错误日志等
- **统计数据**：销售统计、用户统计等

### 2.2 应该放在本地的数据

#### 2.2.1 个人隐私数据

- **日记数据**：智能日记内容、心情、标签等
- **习惯数据**：习惯设置、打卡记录等
- **待办事项**：待办列表、完成状态等
- **个人设置**：应用设置、主题偏好等
- **个人文件**：个人照片、录音、视频等

#### 2.2.2 缓存数据

- **商品缓存**：浏览过的商品信息
- **图片缓存**：商品图片、用户头像等
- **搜索历史**：搜索关键词、浏览历史等
- **临时文件**：编辑中的文件、上传中的文件等

#### 2.2.3 离线数据

- **离线商品**：下载的商品信息
- **离线订单**：历史订单缓存
- **离线日记**：本地日记备份

## 3. 数据同步策略

### 3.1 实时同步

- **用户认证数据**：登录、注册、修改密码等
- **订单数据**：创建订单、支付状态等
- **商品数据**：商品信息、库存更新等
- **信用积分**：积分变动、等级变更等

### 3.2 定期同步

- **用户扩展数据**：个人资料更新
- **日记数据**：定期备份到云端
- **习惯数据**：定期同步打卡记录
- **统计数据**：销售数据、用户行为数据等

### 3.3 手动同步

- **大文件**：视频、高清图片等
- **批量数据**：批量商品导入、批量订单处理等
- **备份数据**：手动备份本地数据到云端

## 4. 数据安全策略

### 4.1 服务器数据安全

- **数据加密**：敏感数据加密存储
- **访问控制**：严格的权限管理
- **备份策略**：定期备份服务器数据
- **日志审计**：记录所有数据操作

### 4.2 本地数据安全

- **本地加密**：本地数据库加密
- **访问保护**：应用锁、生物识别
- **数据清理**：定期清理缓存数据
- **隐私设置**：用户可控制数据同步范围

## 5. 数据迁移策略

### 5.1 从本地到服务器

- **用户数据迁移**：将本地用户数据同步到服务器
- **商品数据迁移**：将本地商品信息上传到服务器
- **订单数据迁移**：将本地订单同步到服务器

### 5.2 从服务器到本地

- **离线数据下载**：下载必要数据到本地
- **缓存数据更新**：定期更新本地缓存
- **恢复数据**：从服务器恢复本地数据

## 6. 存储优化策略

### 6.1 服务器存储优化

- **数据压缩**：压缩存储大文件
- **数据分片**：按时间或类型分片存储
- **过期清理**：定期清理过期数据
- **CDN加速**：使用CDN加速静态文件

### 6.2 本地存储优化

- **增量同步**：只同步变更数据
- **智能缓存**：根据使用频率调整缓存策略
- **本地压缩**：压缩本地存储的文件
- **清理策略**：自动清理临时文件

## 7. 实施建议

1. **创建cloud目录结构**：在用户设置目录下创建cloud目录及其子目录
2. **修改数据库配置**：将用户和电商相关数据库指向cloud目录
3. **更新存储服务**：修改StorageService以支持服务器文件存储
4. **实现数据同步**：开发数据同步模块，支持实时和定期同步
5. **添加安全措施**：实现数据加密和访问控制
6. **提供用户选项**：允许用户控制数据存储位置和同步策略
7. **测试验证**：确保数据在本地和服务器之间正确同步

## 8. 总结

通过明确区分服务器数据和本地数据，可以：

- **提高数据安全性**：敏感数据集中管理，个人隐私数据本地存储
- **提升系统性能**：本地数据快速访问，服务器数据集中处理
- **增强用户体验**：支持离线使用，数据实时同步
- **优化存储资源**：合理分配存储空间，减少不必要的同步

此策略既满足了电商系统的集中管理需求，又保护了用户的个人隐私，同时支持离线使用场景。
